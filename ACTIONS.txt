Claude Code DevTools - action reference
=======================================

The assistant can use these tags in replies. The DevTools panel parses them,
runs the requested page or file action, and sends the result back into the chat.

[ACTION:eval]JavaScript code[/ACTION]
    Run JavaScript in the inspected page and return the result.
    Example: [ACTION:eval]document.querySelectorAll('a').length[/ACTION]

[ACTION:dom]CSS selector[/ACTION]
    Return the first matched element's outerHTML.
    Example: [ACTION:dom]#main[/ACTION]

[ACTION:dom:all]CSS selector[/ACTION]
    Return all matched elements' outerHTML.
    Example: [ACTION:dom:all].item[/ACTION]

[ACTION:text]CSS selector[/ACTION]
    Return visible text for the matched element.
    Example: [ACTION:text]h1[/ACTION]

[ACTION:click]CSS selector[/ACTION]
    Scroll the matched element into view and click it.
    Example: [ACTION:click]button[type="submit"][/ACTION]

[ACTION:input]CSS selector
text to type
[/ACTION]
    Focus the matched input-like element, set its value, and dispatch input/change events.
    Example:
    [ACTION:input]input[name="country"]
    Singapore
    [/ACTION]

[ACTION:press]key[/ACTION]
    Dispatch keydown/keyup events on the active element.
    Example: [ACTION:press]Enter[/ACTION]

[ACTION:console][/ACTION]
    Return recent console logs.

[ACTION:network][/ACTION]
    Return recent network requests.

[ACTION:title][/ACTION]
    Return the page title.

[ACTION:url][/ACTION]
    Return the current page URL.

[ACTION:copy]content[/ACTION]
    Copy content to the system clipboard.
    Example: [ACTION:copy]console.log('hello')[/ACTION]

[ACTION:project:scan][/ACTION]
    Scan the local frontend project under the bridge write root. The report
    includes framework, bundler, package manager, package scripts, config files,
    key directories, dependency hints, likely entry files, and data/service
    candidates.

[ACTION:file:list]glob pattern[/ACTION]
    List local project files under the bridge write root.
    The bridge filters .git, node_modules, dist, build, and __pycache__.
    Example: [ACTION:file:list]*countr*[/ACTION]

[ACTION:file:read]path[/ACTION]
    Read a local project file under the bridge write root.
    Example: [ACTION:file:read]src/data/countries.json[/ACTION]

[ACTION:save]path
file content from the next line
[/ACTION]
    Write a local file under the bridge write root and create parent directories.
    The default write root is the directory where the bridge was started.
    You can override it with CC_DEVTOOLS_WRITE_ROOT.
    Example:
    [ACTION:save]public/cc-devtools/countries.json
    [{"code":"SG","name":"Singapore"}]
    [/ACTION]
