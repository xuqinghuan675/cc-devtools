================================================================
  Claude Code DevTools — 使用说明
================================================================

这个工具让你在 Chrome 浏览器按 F12 打开的面板里直接和 Claude Code 对话，
CC 能读取网页内容，也能通过你的指令操作网页。


================================================================
一、这个文件夹里有什么
================================================================

  D:\F12\
  ├── start-bridge.bat    ← 每次用之前双击它
  ├── install.bat         ← 只运行一次，装依赖用的（你已经装过了）
  ├── ACTIONS.txt         ← CC 支持的所有操作命令
  ├── 使用说明.txt         ← 就是你正在看的这个文件
  ├── bridge/             ← 桥接程序（连接 CC 和浏览器）
  └── extension/          ← Chrome 扩展（F12 面板本身）


================================================================
二、启动（每次使用都要做）
================================================================

第1步：双击 D:\F12\start-bridge.bat

  会弹出一个黑底白字的命令行窗口，显示：

    CC DevTools Bridge Server
    ws://localhost:9876

  看到这行字就说明启动成功了。
  这个窗口不要关，让它一直开着。关了就用不了。
  如果窗口显示"端口 9876 已被占用"，说明上次的还没关，找到之前的黑窗口关掉再重试。

第2步：打开 Chrome 浏览器，加载扩展

  这一步只需要做一次，以后不用重复：

  2.1 在 Chrome 地址栏输入：chrome://extensions  然后回车
  2.2 页面右上角有一个开关叫"开发者模式"，把它打开
  2.3 页面左上角会出现三个按钮，点击"加载已解压的扩展程序"
  2.4 在弹出的文件夹选择窗口中，找到 D:\F12\extension 文件夹，点"选择文件夹"
  2.5 扩展列表里会出现一个叫"Claude Code DevTools"的项目，说明加载成功

第3步：打开一个网页，按 F12

  3.1 打开你想操作的网页（比如百度、B站，随便什么）
  3.2 按键盘上的 F12 键（通常在键盘最上面一排）
  3.3 浏览器右侧或底部会弹出开发者工具面板
  3.4 在面板顶部的标签栏里找到"Claude Code"（可能需要点 >> 箭头展开更多标签）
  3.5 点击"Claude Code"标签


================================================================
三、界面说明
================================================================

进入 Claude Code 面板后，你会看到：

  ┌─────────────────────────────────────────┐
  │ 未连接    │                    [采集] [?] [重置] │  ← 顶部状态栏
  ├─────────────────────────────────────────┤
  │                                         │
  │           （聊天消息区域）                 │
  │                                         │
  ├─────────────────────────────────────────┤
  │ [输入框                        ] [发送] │  ← 底部输入区
  └─────────────────────────────────────────┘

  状态指示灯：
    红色"未连接"  → Bridge Server 没启动，去双击 start-bridge.bat
    绿色"已连接"  → 一切正常，可以开始用

  按钮：
    [采集]  把当前网页的内容（标题、文字、结构）发送给 CC，让 CC"看到"这个页面
    [?]     显示 CC 能执行的操作列表
    [重置]  清空所有对话记录，从头开始

  输入框：
    在这里打字，然后按 Ctrl+Enter 发送（不是直接按 Enter）


================================================================
四、一个完整的操作示例
================================================================

假设你想让 CC 帮你分析百度首页：

  1. 打开 www.baidu.com
  2. 按 F12 → 点击"Claude Code"标签
  3. 确认左上角显示绿色"已连接"
  4. 点击 [采集] 按钮
     → 聊天区会出现一行黄色小字"页面内容已采集"
  5. 在输入框输入：这个页面是做什么的？有哪些主要功能？
  6. 按 Ctrl+Enter 发送
     → CC 会回复，列出页面的标题、主要按钮、搜索框等信息
  7. 你可以继续问：帮我点一下搜索按钮
     → CC 可能会回复：[ACTION:eval]document.querySelector('按钮选择器').click()[/ACTION]
     → 面板会自动执行这个 JS 代码，在页面上点击按钮

这就是一个完整的对话流程：采集页面 → 提问 → CC 分析 → CC 执行操作。


================================================================
五、CC 能做什么
================================================================

CC 通过 [ACTION:xxx] 标签来操作网页，你不需要自己写这些标签，
CC 会根据你的问题自动生成。你能做的就是像聊天一样提问。

你可以让 CC 做的事：

  查信息：
    "这个页面的标题是什么"
    "找出页面上所有的链接"
    "页面上有哪些按钮"
    "这个 div 里面有什么内容"

  改页面：
    "把页面背景改成黑色"
    "隐藏那个弹窗广告"
    "把标题改成 Hello World"
    "给所有图片加个红框"

  查数据：
    "页面加载了哪些 JS 文件"
    "有没有报错，看一下控制台"
    "最近发了哪些网络请求"
    "页面上有多少张图片"


================================================================
六、常见问题
================================================================

Q: 双击 start-bridge.bat 闪一下就没了
A: 说明之前的 Bridge Server 还在运行，检查任务栏有没有黑窗口。
   或者按 Ctrl+Shift+Esc 打开任务管理器，找到 node.exe 进程结束掉再重试。

Q: 面板一直显示红色"未连接"
A: 1) 确认 start-bridge.bat 的黑窗口还开着
   2) 确认黑窗口里显示的是"运行在 ws://localhost:9876"
   3) 如果窗口显示"端口已被占用"，关掉所有黑窗口重试

Q: CC 回复的内容看不懂
A: 正常，有时候 CC 会先自动执行 [ACTION:url] [ACTION:title] 来获取页面信息，
   稍等一下面板会自动处理，然后 CC 会给出新的回复。

Q: 采集按钮点了没反应
A: 当前页面可能没有加载完成，刷新页面后再试。或者页面是空白页（如 chrome://extensions），
   换个普通网页再试。

Q: CC 回复"未找到元素"
A: CC 用的 CSS 选择器没匹配到页面元素。告诉 CC"没找到，换个方式试试"即可。

Q: Ctrl+Enter 发送不了
A: 有些输入法会拦截快捷键，试试先按 Shift 切到英文输入法，再按 Ctrl+Enter。
   或者直接鼠标点 [发送] 按钮。
